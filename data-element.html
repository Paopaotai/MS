数据元件加工工序

1

数据汇聚治理

用DataClean导入原始数据，执行缺失值补全、异常值剔除，生成清洗报告

关键控制点：缺失率≤10%，敏感字段需脱敏

2

元件加工

FeatureEngine训练模型，SecurePack封装元件，生成元件ID

关键控制点：模型准确率≥85%，封装需包含溯源码

3

检测

ElementValidator校验标准合规性，SecurityScanner检测安全漏洞

关键控制点：需通过标准+安全+合规三重检测

4

流通

用ValueMetric计算估值，区块链存证权属信息，推送至数据要素互联网

关键控制点：估值需标注计算逻辑，存证信息不可篡改

5

使用

应用方通过API调用，MeteringKit计量调用次数，记录上链

关键控制点：调用需验证权限，计量误差≤1%

API调用示例

# 企业处罚风险指数元件调用示例
url = "https://element.secretflow.com/api/v1/yy-fin-risk-index"
headers = {
    "Authorization": "Bearer YOUR_DEVELOPER_TOKEN",
    "Content-Type": "application/json"
}
params = {
    "enterprise_id": "91110108MA01234567",
    "time_range": "202401-202410"
}

# 响应结果
response = {
    "code": 200,
    "data": {
        "risk_index": 0.72,
        "risk_detail": "2次行政处罚，金额50万元",
        "trace_id": "YY-Fin-Risk-20241028-001"
    }
}

常见问题解答

中小机构无数据金库，如何使用隐语元件？

可租用隐语社区"公共数据金库"，或通过ElementKit对接第三方金库

元件跨区域流通需哪些合规材料？

需提供合规检测报告、区块链确权证明，适配目标区域地方标准

隐语元件与其他平台元件能否互通？

支持通过InterConnector工具转换格式，确保跨平台流通
